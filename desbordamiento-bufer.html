2.3.14 Desbordamiento de búfer

Cuando un programador crea un programa tiene que controlar el tamaño del buffer destinado para dicho programa, o sea tiene que controlar la cantidad máxima de datos que se puede copiar en el área de memoria reservada para dicha aplicación. Algunos programadores se olvidan de contemplar esto cuando realizan sus aplicaciones convirtiéndose en una vulnerabilidad de software donde la cantidad de datos a almacenar es superior al tamaño del buffer. Y esto es una vulnerabilidad porque no hay un control sobre los bytes restantes al tamaño del buffer (tope del buffer); o sea los bytes sobrantes se van a almacenar en zona perteneciente a otros datos de utilidad o código almacenado en dicha memoria. O sea se pisan datos que pueden ser útiles o se pisa código que justo se necesita para ese momento. Esto constituye un fallo de programación. El mismo puede ser aprovechado por un atacante para influir sobre el funcionamiento del sistema.

Se denomina shellcode al código ejecutable especialmente preparado que se copia al host objeto del ataque para obtener los privilegios del programa vulnerable ya que si el programa con el error en cuestión tiene privilegios especiales constituye en un fallo grave de seguridad.
