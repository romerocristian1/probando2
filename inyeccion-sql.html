2.3.13 Inyección SQL

SQL o lenguaje de consulta de datos es un lenguaje utilizado en las base de datos relacionales. En los servidores web se usa este lenguaje para acceder a base de datos.

El ataque por inyección de SQL se produce cuando no se produce un buen filtro de la información enviada por el usuario. Ya que un usuario malicioso puede introducir y ejecutar sentencias SQL que el servidor no debería porque aceptar; independiente del sistema de base de datos, ya que depende solo de una inapropiada o falta de validación de los datos de entrada.

Se dice que existe o se produjo una inyección SQL cuando, de alguna manera, se inserta o "inyecta" código SQL invasor dentro del código SQL programado, a fin de alterar el funcionamiento normal del programa y lograr así que se ejecute la porción de código "invasor" incrustado, en la base de datos.

La vulnerabilidad se puede producir cuando una persona inyecta código sql indebido y el sistema no cuenta con la programación necesaria para evitar que esto suceda.

Al ejecutarse la consulta en la base de datos, el código SQL inyectado también se ejecutará y podría hacer un sinnúmero de cosas, como insertar registros, modificar o eliminar datos, autorizar accesos e, incluso, ejecutar otro tipo de código malicioso en el sistema informático.
