2.3.11 Rookits

Son programas que ocultan procesos, archivos o entradas del Registro de sistemas. Dicho software no es malicioso en sí, pero es utilizado por los creadores de malware para esconder evidencias en los sistemas infectados. O sea permite que el malware permanezca oculto al usuario. Entonces por medio de los rootkits un atacante evita que un malware sea visible en la lista de procesos del sistema o que sus ficheros sean invisibles en el explorador de archivos. Este tipo de comportamiento permite la ocultación de cualquier indicio de que el ordenador está infectado por un malware.

Para lograr esto, los Rookits trabajan como parte del sistema operativo así de esta manera logran ocultar a los usuarios las tareas o servicios reales del malware. Por consiguiente, se conseguir que el sistema operativo esté bajo el control total del atacante. Rootkits tienen dos grupos principales con diferentes arquitecturas, Rootkits clásica y rootkits de kernel.

Un rootkit ataca directamente el funcionamiento de un sistema operativo. En Windows, se interceptan los APIs (Interfaz de Aplicaciones de Programación) del sistema operativo. Mientras que en Linux, se modifican para trabajar directamente en el kernel del sistema. Estas, interactúan entre el usuario y el kernel del sistema operativo.

Los rootkits los utilizan los cibercriminales con el propósito de realizar delitos informáticos para conseguir beneficios económicos y pasar de manera inadvertida ante las organizaciones atacadas. De esta forma el malware esta la mayor de tiempo posible en el sistema de la víctima sin ser detectado.

Algunos programas malignos también contienen rutinas para evitar ser borrados, y no solo para ocultarse.
